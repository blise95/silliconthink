## ADDED Requirements

### Requirement: 公开列表加载失败须展示错误而非空列表
公开博客列表页 SHALL 在内容接口失败时展示明确错误信息（及可选重试）；SHALL NOT 在存在加载错误时仅显示「没有文章」类空状态。

#### Scenario: 接口失败显示错误
- **WHEN** 公开文章列表 API 返回非成功（网络错误或业务错误）
- **THEN** 页面展示错误提示，且不渲染为「正常的空列表」

#### Scenario: 真正无文章时仍可空状态
- **WHEN** 列表 API 成功且 `list` 为空
- **THEN** 页面可展示空状态提示

### Requirement: 作者保存须有失败与成功反馈
作者写作台在保存草稿或发布时，SHALL 在失败时展示后端或校验错误信息；SHALL 在成功时提供可感知的成功反馈（不仅是无提示的静默完成）。

#### Scenario: 存储不可用时保存失败可见
- **WHEN** 对象存储根不可写导致创建或更新失败
- **THEN** 写作台展示与存储不可用相关的错误信息，用户可感知操作未成功

#### Scenario: 保存成功有反馈
- **WHEN** 作者成功保存草稿
- **THEN** 界面给出短暂或持久的成功提示

### Requirement: 正文加载在对象缺失时回退遗留字段
系统在按 `content_key` 读取正文对象失败（文件不存在）时，SHALL 若元数据中仍存在非空遗留 `content_md` 则使用该字段返回正文；若两者皆不可用，SHALL 返回可理解的业务错误而非静默空正文冒充成功（针对需要正文的详情接口）。

#### Scenario: 对象缺失但库内有遗留正文
- **WHEN** 详情请求对应文章的 `content_key` 文件不存在，且 `content_md` 非空
- **THEN** 响应仍包含可用的 `contentMd` 正文

#### Scenario: 对象与遗留字段皆无
- **WHEN** 详情需要正文，且对象缺失且无可用遗留 `content_md`
- **THEN** 系统返回明确的内容缺失类错误

### Requirement: 写入前校验对象存储可写
系统在创建或更新文章正文前 SHALL 确认对象存储根可写；不可写时拒绝写入并返回存储不可用类错误。

#### Scenario: 不可写时创建失败
- **WHEN** 存储根不可写且作者尝试创建文章
- **THEN** 系统不留下成功写完的权威正文，并返回存储不可用错误
