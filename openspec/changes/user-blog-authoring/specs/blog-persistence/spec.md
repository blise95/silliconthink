## ADDED Requirements

### Requirement: 博客文章持久化与审计字段
系统 SHALL 在 MySQL 中提供博客文章及相关标签表，且每张业务表包含 `create_date`、`update_date`、`create_by`、`update_by`。

#### Scenario: 建表包含文章与标签
- **WHEN** 执行项目提供的 DDL 初始化脚本
- **THEN** 数据库中存在文章主表、标签表与文章-标签关联表，且均含四审计字段

#### Scenario: 写入时维护审计
- **WHEN** 通过应用创建或更新一篇文章
- **THEN** 对应行的创建/更新审计字段被正确赋值

### Requirement: 公开接口仅返回已发布且未删除文章
系统 SHALL 提供公开博客查询 API；仅返回 `published` 且未逻辑删除的文章。

#### Scenario: 公开列表过滤草稿
- **WHEN** 客户端调用公开文章列表接口，且库中存在草稿与已发布文章
- **THEN** 响应仅包含已发布文章

#### Scenario: 公开按 slug 取草稿失败
- **WHEN** 客户端用某草稿的 slug 调用公开详情接口
- **THEN** 系统返回未找到（或不存在等价结果），不返回正文

#### Scenario: 公开支持分页标签与关键词
- **WHEN** 客户端调用公开列表并传入分页、标签或关键词参数
- **THEN** 系统按约定过滤已发布文章并返回分页结构

### Requirement: 作者写接口强制登录与所有权
系统 SHALL 将作者侧创建、更新、发布、取消发布、删除与「我的」查询置于受保护路径；操作目标必须属于当前用户。

#### Scenario: 无令牌无法写
- **WHEN** 未携带有效令牌调用作者创建或更新接口
- **THEN** 系统返回未授权

#### Scenario: 令牌用户只能操作自己的文章
- **WHEN** 用户 A 的令牌用于更新用户 B 的文章 id
- **THEN** 系统拒绝该写操作

### Requirement: slug 全局唯一
系统 SHALL 保证文章 `slug` 在未删除记录中全局唯一；冲突时拒绝写入。

#### Scenario: 重复 slug 创建失败
- **WHEN** 用户尝试创建一篇 slug 与已有未删除文章相同的文章
- **THEN** 系统拒绝创建并返回可理解的业务错误

### Requirement: 标签关联写入
系统 SHALL 在创建或更新文章时接受标签名称列表，维护标签实体与关联关系，并在查询响应中返回标签名称列表。

#### Scenario: 更新标签
- **WHEN** 作者将文章标签从 `["vue"]` 更新为 `["vue", "spring"]`
- **THEN** 后续查询该文章返回的 tags 包含这两个名称

### Requirement: 登录用户可上传图片并获得可访问 URL
系统 SHALL 提供需登录的图片上传接口；校验类型与大小后将文件存入配置的本地目录，并返回可供 Markdown 与封面使用的公开访问 URL。

#### Scenario: 上传成功
- **WHEN** 已登录用户上传合法图片文件（如 png/jpeg，且未超大小上限）
- **THEN** 系统保存文件并返回可公开 GET 访问的 URL

#### Scenario: 非法类型拒绝
- **WHEN** 已登录用户上传非图片或不在允许列表内的文件
- **THEN** 系统拒绝上传且不返回可用 URL

#### Scenario: 公开可读上传文件
- **WHEN** 客户端使用上传返回的 URL 发起 GET
- **THEN** 系统返回对应图片内容（无需登录）
