## Why

站点已具备访客浏览博客与账号登录能力，但文章仍来自前端 Mock，登录用户无法创作内容。现在补齐「登录用户对自己博客的新增、修改与发布」，才能把个人站点从展示页变成可运营的内容站。

## What Changes

- 后端新增博客文章持久化（表结构、公开读、作者写）与所有权校验
- 登录用户可创建草稿、编辑自己的文章、发布 / 取消发布
- 未登录或非作者不可创建 / 修改他人文章；公开列表与详情仅展示已发布文章
- 前端新增作者工作台（我的文章、编辑器、发布操作），并扩展 `PostRepository` 写能力
- 编辑器支持 **分栏实时预览**（左写右看）、工具栏插入常用 Markdown（加粗、代码块等）、粘贴/选择上传图片并插入正文
- 前端视觉升级为 **「墨简」**：乔布斯式极简（留白、克制、字体优先）+ 东方元素（墨色、朱砂点缀、宣纸感背景、标题用衬线中文）
- 内容数据源支持按配置切换：Mock 或真实 API（与现有认证 API 开关模式一致）
- 不引入完整 CMS / RBAC / 评论 / 协作编辑 / 云 OSS

## Capabilities

### New Capabilities
- `blog-authoring`: 登录用户创建草稿、编辑自己的文章、发布与取消发布；分栏实时预览编辑器、工具栏、图片粘贴上传；作者工作台与路由守卫
- `blog-persistence`: 博客表模型、公开已发布内容查询 API、作者侧写 API、图片上传与静态访问、所有权与状态校验

### Modified Capabilities
- `blog`: 公开列表 / 详情在启用 API 时可从后端读取已发布文章；可选展示作者展示名
- `mock-data-layer`: `PostRepository` 扩展作者侧写方法；API 实现与 Mock 写实现可切换
- `public-site`: 全站视觉令牌与关键版式对齐「墨简」语言（去紫系默认风，强化留白与东方气质）

## Impact

- **后端**（`backend/`）：新增 `blog` 包、建表 SQL、图片本地上传与公开访问、Security 放行公开读与保护写/上传
- **前端**（`frontend/`）：作者页、分栏 Markdown 编辑器（工具栏 + 预览）、扩展 `postRepository`、路由与 Header 入口；刷新 `tokens.css` 与公开页/写作页样式
- **依赖**：沿用 Spring Boot + JWT + MyBatis-Plus、Vue 3；编辑器自研轻量组件；字体经 Google Fonts / 系统中文字体栈
- **数据 / 存储**：文章与标签表；上传图片存本地可配置目录并对外提供 URL
- **视觉**：废弃当前 indigo/violet 品牌色，改为墨色 + 朱砂点缀 + 冷宣纸底；写作台与公开站同一套语言
